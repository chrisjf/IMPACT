Text Input Fields

Text input fields are editable by users. They are illustrated as rectangular areas in which users can enter text or modify existing text. Text input fields can be active or disabled. Keyboard focus is supported.

Code snippet

<!-- This code snippet shows how to create Text Input Fields like above: -->
<form action="#">

	<fieldset>
		<!-- Text Field Input needs a wrapping <div> with class "text-input" like this: -->
		<div class="text-input">
			<label for="enabled-text-field1">enabled (default size)</label>
			<input type="text" id="enabled-text-field1" value="Lorem ipsum" />
		</div>
		
		<!-- The size of the Text Field Input can be changed by CSS (stlye="width:200px")-->
		<div class="text-input">
			<label for="enabled-text-field2">enabled (200px)</label>
			<input style="width:200px" type="text" id="enabled-text-field2" value="Lorem ipsum" />
		</div>
		
		<!-- To mark a Text Field Input as readonly the attribute readonly="readonly" has to be added to the <input> -->
		<div class="text-input">
			<label for="readonly-text-field">read only</label>
			<input type="text" id="readonly-text-field" value="Lorem ipsum" readonly="readonly" />
		</div>
	</fieldset>
</form>

Drawbacks

  • There is no disabled state, only the readonly.
  • Selected text in the input is highlighted blue, this color can not be modified.

Buttons

Users can initiate an immediate action with a button, e.g. submit or confirm information. By default, the button in focus is invoked when Enter is pressed. Button labels can be either text, an icon or a combination of both. Buttons can be used singly or in groups (when actions are closely related). Groups of buttons can have a label to easily identify the group.

Single Buttons

Buttons: Input submit buttons

Grouped Buttons

Code snippet

//This java script code snippet shows how to attach a special action to a button:
$(function(){

	//jQuery code to access the button with id "demo-button"
	$("#demo-button").click(function() {

		//Code which is to be executed when the button is clicked.
		alert("You clicked the button...");
		return false;
	});
});
<!-- This HTML code snippet shows how to create buttons like them above. -->
<form action="#">
    <fieldset>
    
        <!-- a button can just be a <button> or a <input type="submit">.
             There is no special class to set and nothing special to be considered.-->
        <button id="demo-button">A button element</button>
        <button disabled="disabled">A disabled button element</button>
        <input type="submit" value="A submit button" />
        <input type="submit" value="A disabled submit button" disabled="disabled" />
    </fieldset>
    
    <!-- For grouped buttons, the sourrounding <fieldset> or <div> must have the class "button-group" -->
    <fieldset class="button-group">
        <button>A button element</button>
        <button>Another button element</button>
    </fieldset >
</form>

Drawbacks

Toggle Buttons

Toggle buttons represent a setting with two states: on and off. Users can use toggle buttons to choose between these two states. Like all buttons, the icon and/or text remains the same whether the button is in the on or off state. Toggle buttons can be placed as single buttons or grouped.
Variation a – multiple choice possible:
more than one toggle button in a group can have the state on (as with check boxes).
Variation b – single choice possible:
only one toggle button in a group can have the state on (as with radio buttons).
Although check boxes/radio buttons and toggle buttons have the same function, they differ in the way they are integrated within the IMPACT toolbox: check boxes and radio buttons are used in dialogue boxes and menus, toggle buttons are used within toolbars or visualisations. Toggle buttons are also used in dialogue boxes if consistency with a toolbar is necessary.
Groups of toggle buttons can have a label to easily identify the group.

Single Toggle Button

Variation a (like checkboxes)

Variation b (like radio buttons)

Disabled Toggle Button

Code snippet

<!-- This HTML code snippet shows how to create toggle buttons like them above. -->
<form action="#">
	<fieldset>
		<!-- A simple Toggel  Button ist just a <input> with type="checkbox" and class="toggle-button" -->
		<!-- To mark a Toggle Button as checked the attribute checked="checked" must be set to the <input>.-->
		<input type="checkbox" class="toggle-button" id="button-as-toggle-button" />
		<label for="button-as-toggle-button">Toggle Me!</label>
	</fieldset>
	
	<!-- For grouped toggle buttons, the sourrounding <fieldset> or <div> must have the class "toggle-button-group" -->
	<!-- To act like check-boxes, the sourrounding <fieldset> must have the class "checkbox-toggle-buttons" -->
	<fieldset class="checkbox-toggle-buttons toggle-button-group">
		<input type="checkbox" id="check1" /><label for="check1">Check 1</label>
		<input type="checkbox" id="check2" checked="checked" /><label for="check2">Check 2</label>
		<input type="checkbox" id="check3" /><label for="check3">Check 3</label>
	</fieldset>
	
	<!-- For Toggel Buttons that act like radio buttons, the sourrounding <fieldset> or <div> must 
	have class "checkbox-toggle-buttons" and in addition the type hast to be "radio" instead of "checkbox" -->
	<fieldset class="checkbox-toggle-buttons toggle-button-group">
		<input type="radio" id="radio1" name="radio" /><label for="radio1">Choice 1</label>
		<input type="radio" id="radio2" name="radio" checked="checked" /><label for="radio2">Choice 2</label>
		<input type="radio" id="radio3" name="radio" /><label for="radio3">Choice 3</label>
	</fieldset>
	
	<fieldset>
		<!-- To disable a Toggel Button add the attribute disabled="disabled" to the input -->
		<input disabled="disabled" type="checkbox" class="toggle-button" id="disabled-toggle-button" />
		<label for="disabled-toggle-button">Toggle</label>
		<input disabled="disabled" type="checkbox" class="toggle-button" id="disabled-active" checked="checked"/>
		<label for="disabled-active">Toggle</label>
	</fieldset>
</form>

Drawbacks

  • Again, there is no selector fot the state "focused and hover" or "focused and clicked", because the hover state includes the focus state.

Check Boxes

Check boxes are used when users are supposed to choose an option, e.g. yes/no. The option is put as a statement (rather than a question), the label indicates the selected state. Check boxes are used only to toggle an option on or off or to select or deselect an item. In contrary to radio buttons, check boxes can be used to choose more than than one option, e.g. in a list of options the selection of more than one options is possible.

Enabled

Disabled

Code snippet

	
<!-- This HTML code snippet shows how to make select boxes like them above -->
<form>
	<fieldset>
		
		<!-- A checkbox must be a <input> with type="checkbox" AND class="checkbox" -->
		<input type="checkbox" class="checkbox" id="checkbox1"/><label for="checkbox1">Check 1</label>
		
		<!-- A checkbox can be disabled with attribute disabled="disabled" -->
		<input type="checkbox" class="checkbox" id="checkbox2" disabled="disabled"/><label for="checkbox1">Check 2</label>
		
		<!-- A checkbox can be selected by default with checked="checked" -->
		<input type="checkbox" class="checkbox" id="checkbox3" checked="checked" /><label for="checkbox1">Check 3</label>
	</fieldset>
</form>

Drawbacks

  • Again, there is no selector fot the state "focused and hover" or "focused and clicked", because the hover state includes the focus state.

Radio buttons allow to choose one option only out of a set of mutually exclusive, yet related options. They work as the channel presets on radios do. All options are visible and can be selected with one click only, with many options possible this requires quite some space.

Disabled

Progress Indicator (indeterminate)

A progress indicator visualises the progress of a lengthy operation. It shows that an operation is ongoing. Progress feedback should be given when operations take longer than 1 second.

Code snippet

<!-- This HTML code snippet shows how to make a progress indicator like that above -->
<!-- Something like a button to react is needed... the main thing is done by Java Script -->
<p><button class="progress-indicator-demo">Click me to show the indicator!</button></p>
<!-- This Java Script code snippet shows how to create a progress indicator -->
$(function(){

	//Attach progress indicator creation to the buttion (just for example)
	$('.progress-indicator-demo').click(function() {
		
		//Call busy() on the buttons parent with parameter 'img' pointing to the image to use
		var that = $(this).parent().busy({img : "css/jquery/plugins/images/busy.gif"});
		
		//Do something while display the indicator (for example sleep 2 seconds)
		setTimeout(function() {
		
			//hide the indicator after 2 seconds
			that.busy("hide");
		
		}, 2000);
	});
	
});

Links

Users can use links to navigate to another page, window, or any other item (e.g. definition). A link is text or a graphic that indicates that it can be clicked. Different states indicate if a link has already been visited or not. These states are indicated by colour – an unobtrusive way to reduce visual clutter. When users hover over a link, the colour changes and the pointer shape (arrow) changes to the a hand shape. Text links are the lightest weight clickable controls and can be used to reduce visual complexity.

Example Link

Datepicker

Users can select a specific date and/or time using the visualisation of a calender or clock. Quick links are provided to ensure quick and easy selection, e.g. today, last month, next month, selection of year.

Inline placed Date Picker

Hidden Date Picker (showed, if clicked into the input field)

Hidden Date Picker (showed, if clicked the button right of the input field)

Code snippet

<!-- This HTML code snippet shows how to make Date Pickers like them above -->
<!-- A visible date picker is just a <div> with class="datepicker-inline" -->
<div class="datepicker-inline"></div>

<!-- A text field  under which a datepicker apears ist just a <input> with class="datepicker"-->
<form>
	<fieldset>
		<label>Date:</label>
		<input class="datepicker" type="text" id="datepicker1" />
	</fieldset>
</form>

<!-- A text field with a button, to trigger the datepicker ist just a <input> with class="datepicker-button"-->
<form>
	<fieldset>
	<label>Date:</label>
	<input class="datepicker-button" type="text" id="datepicker2" />
	</fieldset>
</form>
// This Java Script code snippet shows how to manipulate a datepicker
$(function(){

	//Access the datepicker...
	$('#datepicker1').datepicker( 
		
		//Change the date format of the input field...
		"option", "dateFormat", "d. MM yy"
	);

});

Search Box

Users can search for data using a search box. The decision of whether the IMPACT toolbox uses an instant search (results are displayed immediately as user types, search icon is shown as a graphic, not a button) or a regular search (performed when user clicks search button) depends on the complexity and content used for the individual tools.

Code snippet

<!-- This HTML code snippet shows how to make a Search Box like that above -->

	<!-- A search box must be wrapped with a <div> or <fieldset> with class="search-box" -->

		<label for="search">Search</label>
		<input type="text" name="search" id="search" />
		<input type="submit" value="search"/>
	</fieldset>
</form>
